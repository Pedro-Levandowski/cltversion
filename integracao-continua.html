O que é?

A "Integração Contínua" é uma prática de software que requer frequentemente o commit de código para um repositório compartilhado. O commit de código frequentemente possiblita maior detecção de erros e mais cedo, reduzindo a quantidade de código que o desenvolvedor precisará depurar para corrigir esses erros. Atualizações de código frequentes também são de extrema importância para facilitar a mesclagem de alterações de diferentes membros da equipe de desenvolvimento.

Ao fazer o commit do código no repositório, o desenvolvedor pode construir e testar o código continuamente para garantir que o commit não introduza erros. Seus testes podem incluir linters de código, verificações de segurança, cobertura de código, testes funcionais e outras verificações personalizadas.
