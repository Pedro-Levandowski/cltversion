Controle de Versão

Controle de Mudanças Integração Contínua

O que é?

O "Controle de Versão é um sistema que registra alterações em um arquivo ou conjunto de arquivos ao longo do tempo para que seja possível recuperá-lo mais tarde. Um Sistema de Controle de Versão é uma ferramenta muito importante de ser usada, já que permite que você reverta determinados arquivos à uma versão anterior desejada, ou até mesmo o projeto inteiro, assim como comparar as mudanças realizadas ao longo do tempo de vida dos arquivos ou projeto. Também é possível ver qual pessoa fez determinada modificação que possa ter causado problema no projeto e quando. Usar este tipo de sistema ajuda a ter maior segurança dentro do projeto, já que mudanças críticas podem ser facilmemte revertidas.

Sistemas de Controle de Versão Local

Esse tipo de sistema de controle mantém possuem um simples banco de dados que mantém todas as alterações nos arquivos sob controle e revisão.

Uma das ferramentas mais famosas é um sistema chamado "RCS" que ainda hoje é distribuido com muitos computadores. Essa ferramenta funciona mantendo conjunto de "patches" (diferenças entre arquivos) em um formato especial no disco, assim podendo recriar a aparência de qualquer arquivo em qualquer período do tempo, juntando todos os patches.

Sistemas de Controle de Versão Centralizado

Esse tipo de sistema foi desenvolvido para solucionar o problema das pessoas precisarem colaborar com desenvolvedores em outros sistemas. Como solução, esses sitemas têm um único servidor que contém todos os arquivos versionados e vários clientes que fazem check-outs dos arquivos daquele local central. Essa configuração oferece muitas vantagens sobre os sistemas locais, já que todos sabem até certo ponto o que todos os outros estão fazendo no projeto, além de que os administradores tem acesso refinado ao controle de quem pode fazer o que, isso é mais fácil do que lidar com bancos de dados locais em cada cliente.

Sistemas de Controle de Versão Distribuídos

Nesse tipo de sistemas os clientes não apenas verificam o snapshot mais recente, mas também espelham completamente o repositório, incluindo seu histórico completo. Isso é importante, pois caso algum servidor "morra" e esses sistemas estavam tendo colaboração por meio desse servidor, qualquer um dos repositórios do cliente podem ser copiados de volta ao servidor.
